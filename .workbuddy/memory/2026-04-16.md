## pickup_sink.py v7.2 往返逻辑修正

### 修复的3个核心Bug
1. **载客量覆盖**: `load = min(bus_capacity, remaining[j])` 忽略已有载客 → `space_left = bus_capacity - cur_load; new_load = min(space_left, remaining[j])`
2. **满载判断**: `is_full = (load >= bus_capacity)` 只看单轮 → `is_full = (total_load >= bus_capacity)` 看累计
3. **巴士在站无处理**: 巴士在站清空/关闭后无出路 → Case A2 直接去避难所

### process() 重写要点
- 引入明确状态机: Case A(巴士在站有载客) + Case B(巴士在depot/shelter/空站)
- Case A: 继续装载(累计载客)或去避难所; Case B: 寻找最佳开放上车点
- 巴士到达避难所后自动成为Case B, 下一轮迭代可被派往开放上车点 → 实现往返
- max_iter: n_buses×20 → n_buses×50 (允许多轮往返)
- 全局可达性检查: 无巴士能到达任何有剩余人口的上车点时才宣告不可疏散
